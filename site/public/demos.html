Faithful-ish recreations of what pibarm's TUI pieces look like in a terminal, so you can poke at them before installing.

Shared task widget

Todos and delegated agents render as compact horizontal pills below pi's editor. Drive it like the todo_list tool would:

pi — task widget
› …conversation above…
type your prompt…▍

In the real widget, pills are plain text like ‹ ○ 1 · inspect auth › laid out by the TUI; /tasks shows the expanded vertical view when pills overflow.

Forge-aware statusline

The footer packs project, model, context usage and thinking level on the left; branch, dirty diff stats, Jira ticket, PR and CI state on the right. Toggle repo states:

pi — statusline
…transcript…

Colour mapping — PR: green open, grey draft, purple merged, red closed. CI: pass, running, failing. Icons are Nerd Font glyphs in the real TUI.

Matrix: WezTerm agent panes

The Matrix flow: spawn visible agents into a session-specific WezTerm workspace, watch them work, then /matrix-join to collect logs and clean up.

wezterm — workspace: matrix-myapp-a1b2c3d4
┌─ pane 1 · matrix-scout ─────────
(empty)
┌─ pane 2 · matrix-planner ───────
(empty)
pi — parent session
› waiting…

Real agents run pi -p non-interactively in each pane, stream a start/log banner, write logs to .pi/matrix/, and exit when done. matrix_join waits on their status files, returns the logs to the parent, and closes the panes.
