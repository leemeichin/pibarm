πbarm

Extensions and skills for the pi coding agent: plan first and edit later, run risky work in git worktrees, spawn visible WezTerm agent panes, delegate to headless subagents, and keep repo/PR/CI status one glance away.

See the demos Command reference Tool reference

Quick start

Install into any project pi trusts:

pi install git@github.com:leemeichin/pibarm.git

Or clone for local development:

git clone git@github.com:leemeichin/pibarm.git
cd pibarm
bun install
bun run setup   # copies config examples + runs the setup doctor
pi

What's inside

Plan-first workflow

/plan enters read-only plan mode, asks clarifying questions with rich TUI inputs, and requires approval before anything is edited.

Worktree isolation

/execute-plan worktree <name> runs changes in a repo-local git worktree under .pi/wt/, keeping your active checkout untouched.

Matrix agent panes

Parent-controlled agents in visible WezTerm splits/tabs: /matrix, /matrix-spawn, /matrix-join. Watch scouts and planners work in real time.

Headless subagents

run_subagent / run_subagents run isolated pi -p jobs — in parallel, optionally across different models.

Watcher agents

watch_agent starts a sibling process that polls a PR (or any command) and runs a headless pi task only when the observed state changes.

MCP via mcporter

mcporter_list / mcporter_call bridge MCP servers through the mcporter CLI with configurable command templates.

Forge-aware statusline

An icon-first footer with project, model, context usage, branch, dirty diff stats, PR state and CI state — GitHub via gh, SourceHut via hut.

Shared task widget

Todos and delegated agents render as compact horizontal pills below the editor, so parallel work stays visible without a tall list.

Model presets

/preset planner, /preset executor — switch provider/model, thinking level, and active tools by role.

The main workflow

pi — plan → approve → worktree
› /plan add rate limiting to the API client pi inspects read-only and asks questions if needed… ? Scope: which endpoints need limiting? [all] [mutations only] [custom] you approve or refine the plan… › /execute-plan worktree rate-limit ✓ worktree .pi/wt/rate-limit on branch pibarm/rate-limit changes happen in the worktree, not your active checkout › /worktree-diff .pi/wt/rate-limit +184 −12 across 6 files — review, merge, or abandon

External tooling

pibarm itself is a pi package, but some features shell out to local CLIs. bun run doctor checks your setup:

ToolEnables
piloads this package and runs subagents (required)
gitrepo status and worktrees (required)
weztermMatrix visible agent panes
ghGitHub-backed forge_* tools and statusline PR/CI checks
hutSourceHut-backed forge_* tools
mcporterMCP bridge tools
terminal-notifieroptional native macOS notifications

The TUI uses Nerd Font glyphs for the statusline, Matrix/task widget, and rich planning questions — install a Nerd Font if icons render as boxes.
