Skills

Progressive-disclosure workflows: brief top-level instructions the model loads on demand with /skill:<name>, or automatically when the task matches. Each lives in skills/<name>/SKILL.md.

/skill:agent-orchestration

Plan and execute work with dedicated planner, executor, and subagent roles. Delegation guidance for run_subagent(s), run_worktree_agent, watchers, and Matrix — when each is the right tool.

/skill:matrix

Operating guide for the WezTerm cockpit: spawn read-only scouts first, join before acting, worktrees for risky workers, forced cleanup with matrix_kill all.

/skill:plan-worktree

The plan-first workflow: read-only planning, explicit approval, worktree execution, review with summarize_worktree_diff.

/skill:mcporter

Using MCP servers through the mcporter bridge: discovery, schemas, calling tools, reading resources.

/skill:model-presets

Switching model/tool/thinking presets by role with /preset and .pi/agent-presets.json.

/skill:pr-open

Opening pull requests cleanly — newline-safe descriptions, sensible titles, the right base branch.

/skill:pr-review

Reviewing PRs/patches: gathering the diff, structuring findings, posting inline comments on GitHub when possible.

/skill:ci-triage

Triaging failing CI: fetching logs via the forge tools, isolating the failing step, proposing the smallest fix.

/skill:ruby

Ruby conventions for this environment plus how to run the smallest relevant check.

/skill:typescript

TypeScript conventions plus the minimal type-check/test loop.

How skills load

pi discovers skills from installed packages (this repo declares skills/ in its package.json pi section). Invoke one explicitly with /skill:<name>, or let pi surface it when a prompt matches the skill's description. Skills follow pi's progressive-disclosure format — see pi's docs/skills.md for the format itself.
