Tools

Functions pibarm exposes to the model. The agent calls these itself; you can also nudge it ("use run_subagents to compare…").

Interactive questions

question

Ask the user one focused question, optionally with choices, and return their answer.

ParamTypeNotes
questionstringThe question to ask.
optionsarray?{ label, description? } choices to pick from.
allowCustomboolean?Allow a typed custom answer. Default true.

elicit_plan_questions

Ask several planning questions before finalizing/executing a plan, in a tabbed TUI with Nerd Font status icons, option descriptions/previews, and per-question notes.

Questions can be plain strings (free text) or typed objects:

TypeInput
free_textMulti-line editor
select_oneSingle choice from options
select_manyMultiple choices
confirm / booleanYes/no with optional preview of the action
numberNumeric with default, min, max
{
  "questions": [
    { "label": "Scope", "question": "What should change?", "type": "select_many",
      "options": ["UI", "Tool schema", "Docs"] },
    { "label": "Proceed", "question": "Apply schema changes?", "type": "confirm",
      "preview": "Keeps string questions backward-compatible." },
    { "label": "Retries", "question": "How many retries?", "type": "number",
      "default": 2, "min": 0, "max": 5 }
  ]
}

Worktrees

ToolPurpose
create_git_worktreeCreate an isolated repo-local git worktree and branch (.pi/wt/<name>, pibarm/<name>).
summarize_worktree_diffSummarize status/diff for a worktree — used for review before merge/cleanup.
remove_git_worktreeRemove an isolated worktree after confirmation/review.
run_worktree_agentCreate/use a worktree and run pi -p there; simple tasks may use a lighter available model.

Headless subagents

run_subagent

Run one isolated, non-interactive pi -p subagent and return its output. Defaults to a 10-minute timeout.

ParamTypeNotes
promptstringSelf-contained prompt — the subagent cannot see the parent conversation.
modelstring?pi model pattern. Defaults to the active model; simple-scope tasks may be downgraded to a lighter authenticated model.
timeoutMsnumber?Default 600000.

run_subagents

Run up to 4 subagents in parallel, optionally on different models — for comparisons, parallel research, or planner/reviewer/verifier fan-out.

ParamTypeNotes
jobsarray{ name?, prompt, model?, timeoutMs? } per subagent.
timeoutMsnumber?Default timeout applied per subagent.

watch_agent

Start, stop, or list a sibling watcher process that polls external state and runs a headless pi task only when the observed state changes. Primary use case: PR follow-up while the parent session stays active. Logs under .pi/watchers/.

ParamTypeNotes
actionenum?start (default), stop, list.
prstring?PR number/URL — builds a gh pr view watch command automatically.
watchCommandstring?Any shell command whose output is polled for changes.
goal / loopstring?Claude Code-style outcome + per-cycle instructions (legacy task also accepted).
intervalSecondsnumber?Poll interval, default 300.
maxIterationsnumber?Optional stop after N polls.

Matrix (WezTerm panes)

ToolPurpose
matrix_spawnSpawn a parent-controlled pi agent in a WezTerm pane. Params: role, task, optional model, tools, worktree, placement (right / down / tab / window).
matrix_attachOpen or focus the Matrix WezTerm workspace.
matrix_sendSend a message to a still-running Matrix pane.
matrix_captureCapture recent output from panes/logs (role?, lines? default 80).
matrix_joinWait for agents to finish, return logs, clean up panes (role?, timeoutMs? default 10 min, kill? default true).
matrix_listList known Matrix agents and untracked workspace panes.
matrix_killKill one pane or all Matrix workspace panes.

Roles have default toolsets: scout/reviewer are read-focused, planner is read-only + questions, worker gets normal tools. Agents print a start/log banner, write logs under .pi/matrix/, and their panes exit when the task finishes.

MCP bridge (mcporter)

ToolPurpose
mcporter_listDiscover MCP servers/tools through the mcporter CLI.
mcporter_callCall an MCP tool: server.tool selector + JSON args.
mcporter_resourceList/read MCP resources.

Command templates live in .pi/mcporter.json with placeholders {server}, {tool}, {selector}, {argumentsJson}, {schemaFlag}, {uri}. Default call template:

["call", "{selector}", "--args", "{argumentsJson}", "--output", "json"]

Todo tracking

todo_list

Track progress when one prompt contains multiple requested tasks. Items render as pills in the shared task widget — see the widget demo.

ParamTypeNotes
actionenumset, add, done, list, clear.
itemsstring[]?For set/add.
indexnumber?1-based, for done.
